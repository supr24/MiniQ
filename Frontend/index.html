📚 Compiler Rules & Definitions

🔤 MiniQ Language Syntax

LOAD <table>
FILTER <field> <operator> <value>
SELECT <field1>, <field2>, ...
SORT BY <field> [ASC|DESC]
GROUP BY <field>
AGGREGATE <function>(<field>) AS <alias>, ...
LIMIT <number>

🔑 Identifiers

Definition: Names of tables, fields, and aliases

Rules:

  • Start with letter or underscore (_)
  • Contain alphanumeric characters and underscores
  • Case-sensitive
  • Examples: students, employee_count, _temp

📊 Symbol Table

Definition: Data structure tracking all declared entities

Contains:

  • Tables: Loaded table names and their fields
  • Fields: Column names and their source table
  • Aggregates: Aggregate functions and aliases

Purpose: Validate that referenced fields exist before code generation

🔧 Operators

Comparison: > < = >= <= !=
Example: age > 18, salary = 50000

∑ Aggregate Functions

SUM(field)   - Total of numeric values
COUNT(*)     - Number of rows
AVG(field)   - Average of numeric values
MIN(field)   - Minimum value
MAX(field)   - Maximum value

📦 Supported Tables & Fields

students: [id, name, age, marks, department, email, gpa]
sales: [id, product, category, revenue, quantity, date]
products: [id, name, price, quantity, category, supplier]
employees: [id, name, department, salary, email, joining_date]
orders: [id, customer, amount, date, status, product]
customers: [id, name, email, city, phone, country]

✅ Semantic Validation Rules

  • First statement must be LOAD
  • All referenced fields must exist in loaded table
  • Aggregate functions only work with valid field names
  • SORT BY field must exist in selected fields
  • GROUP BY field must exist
